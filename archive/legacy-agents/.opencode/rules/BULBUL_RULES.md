# BULBUL_RULES

- Execution-first: act, then explain briefly.
- Avoid large reports and repeated policy restatements.
- Always inspect existing components before adding new ones.
- Prefer extending current UI over replacing it.
- Reuse Purchase/Sales layouts and project conventions.
- No new cloud, no sync, no external backend.
- Keep files small and focused.
- Use secure IPC, prepared SQL, and local SQLite patterns.
- Run `npm run build` after modifications.
- Preserve current architecture and naming.
