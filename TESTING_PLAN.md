## المرحلة 3: اختبارات سير العمل (E2E / Workflow Tests)

### 3.1 سير عمل البيع الكامل
```
1. تسجيل دخول (admin/admin123)
2. إنشاء زبون جديد
3. إنشاء منتج جديد مع سعر
4. إضافة مخزون للمنتج
5. إنشاء فاتورة بيع (POS) مع الزبون
6. تأكيد الفاتورة
7. التحقق من:
   - نقصان المخزون
   - تحديث الصندوق
   - تحديث دين الزبون (إذا دفع جزئي)
8. إلغاء الفاتورة
9. التحقق من:
   - إرجاع المخزون
   - إرجاع الأموال
   - إرجاع دين الزبون
```

### 3.2 سير عمل الشراء الكامل
```
1. تسجيل دخول
2. إنشاء مورد جديد
3. إنشاء فاتورة شراء
4. تأكيد الفاتورة
5. التحقق من:
   - زيادة المخزون
   - تحديث دين المورد (إذا دفع جزئي)
   - تحديث سعر التكلفة
6. إلغاء فاتورة الشراء
7. التحقق من إرجاع المخزون والأموال
```

### 3.3 سير عمل المرتجعات
```
1. إنشاء فاتورة بيع
2. تأكيد الفاتورة
3. إنشاء مرتجع مبيعات
4. التحقق من:
   - زيادة المخزون
   - تسوية مالية مع الزبون
5. إنشاء مرتجع مشتريات
6. التحقق من:
   - نقصان المخزون
   - تسوية مالية مع المورد
```

### 3.4 سير عمل الصندوق
```
1. فتح الصندوق
2. بيع منتج (نقدي) → زيادة الرصيد
3. إضافة مصروف → نقصان الرصيد
4. إيداع يدوي → زيادة
5. سحب يدوي → نقصان
6. إغلاق الصندوق → التحقق من التوازن
```

### 3.5 سير عمل إدارة المستخدمين
```
1. تسجيل دخول كـ owner
2. إنشاء مستخدم جديد (cashier)
3. تسجيل خروج
4. تسجيل دخول كـ cashier
5. التحقق من الصلاحيات المحدودة
6. محاولة الوصول لصفحة محظورة → رفض
```
