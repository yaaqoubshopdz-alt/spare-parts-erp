# 🏗️ Spare Parts ERP — دليل معمارية النظام | System Architecture Guide

يقدم هذا المستند توثيقاً فنياً شاملاً لبنية ومعمارية تطبيق إدارة قطع الغيار (Spare Parts ERP) لضمان اتساق التطوير وجودة الكود.

---

## 1. ⚙️ التقنيات الأساسية | Tech Stack

* **البيئة الرئيسية**: [Electron](https://www.electronjs.org/) (إصدار 28+) لتوفير بيئة سطح مكتب محلية آمنة.
* **الواجهة الأمامية**: [React](https://react.dev/) (إصدار 18+) مع [TypeScript](https://www.typescriptlang.org/) و [Vite](https://vite.dev/) كأداة بناء وتطوير سريعة.
* **التصميم والتنسيق**: [Tailwind CSS](https://tailwindcss.com/) (إصدار 3.4+) مع نظام متغيرات CSS للثيمات.
* **إدارة الحالة**: [Zustand](https://zustand-demo.pmnd.rs/) لإدارة الحالة المحلية ومستوى التطبيق بمرونة وخفة.
* **قاعدة البيانات**: [SQLite](https://www.sqlite.org/) مدمجة بالكامل ومدارة عبر مكتبة [better-sqlite3](https://github.com/WiseLibs/better-sqlite3) (محلي فقط، بدون مزامنة سحابية).
* **مخطط البيانات (ORM)**: [Drizzle ORM](https://orm.drizzle.team/) لتعريف الجداول وبناء الاستعلامات الآمنة وترقيتها (Migrations).
* **الاختبارات**: [Vitest](https://vitest.dev/) للاختبارات البرمجية السريعة والحديثة.

## 3. 🔒 معايير الأمان وجسر الـ IPC | Security & IPC Bridge

* **عزل كامل للـ Renderer**: يتم إيقاف `nodeIntegration` وتفعيل `contextIsolation` و `sandbox` لحماية المستخدم من أي ثغرات برمجية قد تنفذ أكواداً على نظام التشغيل.
* **جسر preload.ts**: لا يتعامل الـ Renderer مع وظائف Node.js أو قاعدة البيانات بشكل مباشر. التواصل يتم حصرياً عبر `window.electronAPI.invoke(channel, args)`.
* **القائمة البيضاء (IPC Whitelist)**:
  * يجب تسجيل أي قناة IPC جديدة في الثابت `ALLOWED_INVOKE_CHANNELS` داخل [preload.ts](file:///c:/aissa/electron/preload.ts).
  * أي قناة غير مسجلة سيتم رفضها تلقائياً وصامتاً لأسباب أمنية.
* **تسمية القنوات**: تتبع النمط `db:<entity>:<action>` (مثال: `db:products:create`) أو `icount:<action>` لجرد المخزون.

## 4. 🗄️ قواعد وأمان قاعدة البيانات | Database Safety Rules

* **التنفيذ في العملية الرئيسية (Main Process)**: تجرى جميع عمليات قاعدة البيانات في بيئة Node.js (عملية Electron الرئيسية) ولا تُستدعى مكتبة `better-sqlite3` أو تُفتح قاعدة البيانات داخل واجهة React (Renderer).
* **الحماية من حقن SQL (SQL Injection)**:
  * يمنع منعاً باتاً دمج النصوص (String Interpolation أو Concatenation) داخل الاستعلامات.
  * يجب استخدام Prepared Statements وتمرير المتغيرات كمعاملات آمنة (Parameters `?`).
* **استعلامات الترتيب الآمنة (SORT_MAP)**:
  * عند استخدام `ORDER BY` ديناميكي، يجب استخدام نمط `SORT_MAP` لمطابقة اسم العمود المطلوب بجدول حقيقي وتمريره بدقة. يمنع تمرير نصوص حرة مرسلة من واجهة المستخدم مباشرة للـ `ORDER BY`.
* **المعاملات المالية (Transactions)**:
  * العمليات المحاسبية أو المخزنية المتشابكة (مثل إنشاء فاتورة وخصم الكمية وتحديث حساب العميل) يجب إدراجها بالكامل ضمن `db.transaction()` لضمان سلامة البيانات وحمايتها من التلف والتعليق الجزئي.

## 5. 🎨 الواجهات وتنسيق الألوان | Theme & UI Patterns

* **ثيم داكن هادئ (Dark-First)**: يعتمد التطبيق تيمة مظلمة احترافية تستخدم ألواناً رمادية وزرقاء هادئة تتسق مع راحة العين وتصميم صفحة فواتير المشتريات.
* **حظر الألوان البراقة والذهبية**:
  * يمنع استخدام اللون الذهبي (Gold) أو البرتقالي الفاقع أو الأصفر اللامع في أي واجهة أو تدرج لوني.
  * تستخدم درجات الأخضر والأحمر الهادئة للدلالة على الصعود والهبوط والربح والخسارة.
* **متغيرات CSS للألوان**:
  * لا تكتب قيم ألوان صلبة (Hex `#3b82f6` أو `rgb(...)`) في مكونات React.
  * استخدم فئات Tailwind المستندة لمتغيرات CSS المعرفة في [index.css](file:///c:/aissa/src/index.css) (مثال: `bg-primary_blue` أو `text-danger_red`).
* **صفوف التعبئة الوهمية (Filler Rows)**:
  * تحتوي جداول الجرد والمبيعات والمشتريات على منطق يولد صفوفاً فارغة (Filler rows) لتعبئة الجدول حتى ارتفاعه الأقصى، مما يحافظ على ثبات ارتفاع الجدول والمظهر الجمالي حتى عند احتواء الصفحة على عنصر واحد أو عدم وجود نتائج.

## 6. 🧮 الدقة المحاسبية والحسابات | Financial Accuracy

* **تجنب مشاكل الفاصلة العائمة (Floating Point Issues)**:
  * يتعامل النظام مع عملة الدينار الجزائري، ويجب دمج التقريب العشري لخانقتين دائماً.
  * يجب معالجة العمليات المحاسبية باستخدام دالة التقريب المعتمدة `roundTo2()` المستوردة من [calculations.ts](file:///c:/aissa/src/shared/utils/calculations.ts).
  * لا تستخدم دالة `toFixed()` للعمليات الحسابية أبداً؛ بل استخدمها فقط عند عرض الأرقام كنصوص داخل الواجهات.
