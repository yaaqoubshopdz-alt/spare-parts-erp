## 2. 🔄 دورة العمل المعتمدة | Workflow Sequence

### 🔷 قبل البدء بأي مهمة (Pre-Task)
1. يقرأ الوكيل ملفات الذاكرة الرئيسية الستة لفهم السياق العام للمشروع والقيود والمشاكل السابقة.
2. يفحص المكونات القائمة ذات العلاقة بالمهمة لمنع التكرار وتطبيق إعادة الاستخدام.
3. يحدد أثر التغيير على البنية (الاعتماديات الأمنية، سلامة الجداول، معالجة الحسابات).

### 🟢 بعد إكمال المهمة (Post-Task)
1. يتحقق من نجاح بناء وتجميع المشروع برمجياً عبر `npm run build`.
2. يحدث ملف `session-summary.md` مسجلاً المهام المكتملة والتغييرات الحادثة وحالة البناء.
3. يقرر ما إذا كان هناك أي تحديث مطلوب لملفات الذاكرة الأخرى (مثل إضافة تفضيل جديد تم التوافق عليه، أو تسجيل خطأ تم تجنبه).

## 3. 🎯 كفاءة الذاكرة وتجنب التضخم | Token Efficiency

للحفاظ على ذاكرة خفيفة وسريعة التصفح والاستدعاء، يلتزم الوكيل بالمعايير التالية:
* **نصوص موجزة وقوائم نقطية**: يمنع كتابة مقالات أو شرح مفرط؛ يفضل استخدام جمل قصيرة محددة (5-10 كلمات للنقطة).
* **روابط للملفات بدلاً من نسخ الكود**: لا يتم تخزين أكواد كاملة داخل ملفات الذاكرة، بل يكتفى بالإشارة إلى مسار الملف ورابطه المباشر.
* **حظر البيانات المكررة أو الزائدة**: لا يتم تخزين سجل المحادثات (Chat Logs)، أو مخرجات الأخطاء الطويلة، أو الأفكار غير المعتمدة.
* **الدمج والصيانة**: يدمج الوكيل تلقائياً النقاط المتكررة، ويتم أرشفة سجل الجلسات البرمجية القديمة دورياً للحفاظ على حجم ملف التلخيص.
